# Example: Learning System

## User Problem

A student starts homework late, gets stuck, copies assignments, and sleeps too late.

## Problem Type

Learning / Personal Growth.

## System Boundary

Inside:
- student
- homework tasks
- time window
- phone/game distractions
- parent support
- learning device
- task feedback

Outside:
- school curriculum design
- teacher workload
- exam system

## Information Layer

Inputs:
- homework list
- task difficulty
- student's energy level
- where the student gets stuck
- time remaining
- completion status

Missing information:
- exact stuck points
- whether the task is too hard or just delayed
- which subject drains the most energy

## Energy / Resource Layer

Resources:
- attention
- sleep
- emotional energy
- time before bedtime
- willingness to be guided
- parent-child trust

## Feedback Layer

Leading indicators:
- start time
- first task completed
- stuck duration
- number of help requests
- phone interruptions

Lagging indicators:
- sleep time
- homework completion
- next-day class energy
- test performance

## Failure Diagnosis

Likely failures:
- feedback arrives too late
- tasks are too large
- student avoids tasks because failure feels certain
- parent supervision triggers resistance
- phone provides faster reward loop than homework

## Smallest Useful Experiment

Hypothesis:
A smaller first task and immediate progress feedback will reduce avoidance.

Action:
For 7 days, start with a 10-minute easiest task and mark visible completion before harder work.

Success metric:
Start time improves and first task completion rate exceeds 80%.

Failure metric:
Student still refuses to start or gets stuck before 10 minutes.

## Evolution Roadmap

Keep:
- task formats that produce early completion

Remove:
- large undifferentiated homework blocks

Amplify:
- visible progress and short feedback loops

Test next:
- AI-guided stuck-point prompts
