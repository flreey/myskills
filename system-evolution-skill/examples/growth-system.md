# Example: Growth System

## User Problem

How should I promote my paid web tool?

## Problem Type

Growth / Acquisition.

## System Boundary

Inside:
- target users
- high-intent search pages
- communities
- outreach list
- landing pages
- report CTA
- payment flow

Outside:
- large ad budgets
- broad brand awareness
- generic social media attention

## Information Layer

Inputs:
- search queries users already type
- competitor pages
- community questions
- click and conversion data
- cold outreach replies

## Energy / Resource Layer

Resources:
- content production capacity
- founder time
- SEO authority
- domain trust
- ability to answer questions credibly

## Feedback Layer

Leading indicators:
- search impressions
- page visits
- tool starts
- CTA clicks
- replies from outreach

Lagging indicators:
- payments
- repeat visits
- referrals
- subscriptions

## Failure Diagnosis

Possible failures:
- too much generic content
- traffic without buying intent
- targeting users who only need free answers
- weak landing page promise

## Smallest Useful Experiment

Hypothesis:
High-intent product/risk pages will produce more tool usage than generic blog posts.

Action:
Create 20 high-intent landing pages and contact 30 target users with sample reports.

Success metric:
At least 10 tool starts and 3 CTA clicks from targeted traffic.

Failure metric:
Visits but no searches or CTA clicks.

## Evolution Roadmap

Keep:
- pages and channels that produce tool starts

Remove:
- channels that produce shallow traffic only

Amplify:
- pages that produce report CTA clicks

Test next:
- cold outreach to freight forwarders and brokers
